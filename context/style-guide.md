# FlipMyCase.com — Style Guide

## Formatting Rules
- H1: One per page, includes primary keyword
- H2: 4–7 per article; at least 2 include keyword variations
- Paragraphs: 2–4 sentences max
- Code blocks: use fenced markdown ``` for any code or terminal commands
- Lists: use for steps, options, comparisons

## Terminology Preferences
- "text case" not "letter case"
- "slug" not "URL key"
- "developer" not "coder" or "programmer" (broader)
- Use tool names exactly as they appear on the site

## Attribution
- No author names in blog posts
- Footer attribution: "Published by the FlipMyCase team"

## CTA Style
- Inline: "Try it now → [Tool Name]"
- Never use "click here"
- CTA should reference the specific tool being discussed
