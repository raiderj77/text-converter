# FlipMyCase.com — SEO Guidelines

## Primary Content Focus
Text manipulation, case conversion, developer text tools, productivity tools for writers and devs.

## Keyword Strategy
- Target: long-tail "how to [text task]" queries
- Examples: "how to convert text to lowercase", "slug generator for SEO", "remove line breaks from text"
- Avoid: broad keywords with DA 90+ competitors (Wikipedia, MDN)

## Content Types That Work
- "How to" guides tied to specific text problems
- Comparisons: "camelCase vs snake_case — when to use each"
- Tool roundups: "best free text tools for developers"
- Definitions + use cases: "what is a URL slug and why it matters"

## Internal Linking
Always link to the relevant tool page when mentioning a text task.
Example: when writing about slugs, link to /slug-generator.

## Meta Title Format
`[Topic] — Free Online Tool | FlipMyCase`

## Schema
Use HowTo schema for tutorial posts. Use FAQPage schema on any FAQ section.

## GEO (AI Citation) Requirements
- 40–60 word direct answer blocks under every H2
- Define technical terms clearly (LLMs cite definitional content)
- Include structured lists where possible
